## 二、最终玩法方向调整

旧版是偏传统塔防的短关卡结构：

```text
第 1 关：普通兵教学
第 2 关：加入盾兵
第 3 关：加入骑兵
第 4 关：加入弓兵
```

新方向不采用这种“弱关卡、多短关”的结构，而改为：

```text
单局 5~10 分钟
一个强关卡
敌人持续进攻
击杀敌人升级
升级出现三选一奖励
通过奖励获得字、成语、字池容量、战斗袋容量和构筑强化
```

核心体验应当是：

```text
开局只有基础成语
↓
依靠基础成语击杀敌人
↓
击杀升级
↓
三选一奖励
↓
获得新字 / 完整成语 / 字袋强化
↓
构筑逐渐成型
↓
后期敌人强压
↓
检验玩家本局成语构筑
```

## 三、单局时长与暂停规则

### 1. 单局时长

目标单局时长：

```text
5~10 分钟
```

不做很多短关卡，而是在一局内完成：

```text
开局求生 → 构筑成型 → 压力上升 → 最终强压
```

### 2. 奖励选择时时间暂停

玩家升级后进入奖励三选一界面时，游戏时间必须暂停：

```text
敌人暂停
刷怪暂停
字块补给暂停
弓兵射击暂停
技能表现暂停或等待结算完成
```

选择奖励后恢复战斗。

---

## 四、成语不设置冷却

新规则：

```text
成语不用设置冷却
```

节奏不靠技能冷却限制，而靠下面这些系统自然形成限制：

```text
字从成语字池进入战斗袋需要时间
战斗袋容量有限
成语字池容量有限
字被使用后才回到成语字池
玩家需要在有限字袋中拼出目标成语
```

也就是说：

```text
不是“技能冷却”
而是“字流转冷却”
```

这样更符合成语塔防的核心特色。

## 五、三个核心字系统

后续需要明确分成三层：

```text
1. 成语字池
2. 战斗袋
3. 成语槽
```

### 1. 成语字池

成语字池表示玩家本局已经拥有的字副本。

例如开局不是只有 4 个字副本，而是：

```text
百 x2
步 x2
穿 x2
杨 x2
```

也就是：

```text
百、百、步、步、穿、穿、杨、杨
```

### 2. 战斗袋

战斗袋是当前战斗中玩家可以直接操作的字。

初始建议：

```text
战斗袋容量：6
```

开局应直接装满，不让玩家等第一发技能。

例如：

```text
百、步、穿、杨、百、步
```

### 3. 成语槽

成语槽保持 4 个槽位：

```text
拖入 4 个字 → 检查是否组成已解锁成语 → 成功则释放
```

释放成功后，用掉的字回到成语字池。

---

## 六、字的流转规则

核心流转：

```text
成语字池 → 随时间进入战斗袋
战斗袋 → 拖入成语槽
成语释放成功 → 使用的字回到成语字池
```

### 1. 字只有“使用掉”才回到成语字池

战斗袋中的字不会自动回池。

它需要被：

```text
用于成语释放
或主动弃字
```

才会回到成语字池。

### 2. 弃字也算一种使用

为了避免战斗袋被无用字卡死，后续应设计“弃字”功能：

```text
玩家可以丢弃一个战斗袋里的字
该字回到成语字池
短时间内不会立刻重新进入战斗袋
```

建议第一版规则：

```text
每 3 秒可弃 1 个字
弃字后该字 1.5 秒内不会重新进入战斗袋
```

这样既避免卡死，又不会变成无限洗牌。

## 七、开局节奏设计

### 1. 开局默认成语

开局只给一个基础单体伤害成语：

```text
百步穿杨
```

定位：

```text
单体伤害
优先攻击最靠近城门的敌人
伤害中等
不设置冷却
靠字流转决定释放频率
```

### 2. 开局字种和字副本

开局只有 4 个字种：

```text
百、步、穿、杨
```

但每个字给 2 个副本：

```text
百 x2
步 x2
穿 x2
杨 x2
```

原因：如果开局每字只有 1 个副本，释放一次 `百步穿杨` 后，4 个字都要重新从成语字池慢慢流回战斗袋，节奏会太慢，玩家会陷入“等字”而不是“操作”。

### 3. 开局补字速度

第一版建议：

```text
补字间隔：0.55 秒 / 字
开局前 45 秒可以略快
```

释放一次 `百步穿杨` 后，4 个字重新补齐理论需要：

```text
0.55 × 4 = 2.2 秒
```

加上玩家拖字操作，基础技能大约 3~4 秒一发，节奏比较合适。

### 4. 敌人开局保护

不建议只靠减慢敌人解决开局问题。正确做法是：

```text
开局战斗袋直接装满
基础字每个给 2 个副本
前期补字速度略快
前 30~45 秒敌人速度稍慢
前 5 个敌人血量较低
```

这样玩家一开始有稳定操作，而不是单纯等字。

## 八、奖励三选一规则

### 1. 每次升级给 3 个奖励

升级来源：击杀敌人。

每次升级后暂停时间，弹出三选一奖励。

### 2. 每组三选一至少有 1 个字 / 成语相关奖励

不能出现三张纯数值奖励。

错误示例：

```text
城门血量 +2
补字速度 +10%
敌人金币 +1
```

正确示例：

```text
获得字：万
战斗袋容量 +1
城门血量 +2
```

### 3. 出现的字可以重复出现

新规则：

```text
奖励里的字可以重复出现
```

如果玩家已经拥有某个字，再次选择该字，则表示该字副本 +1。

例如：

```text
已拥有：万 x1
再次选择：获得字「万」
结果：万 x2
```

重复字不是废奖励，而是字袋权重构筑。

## 九、奖励稀有度规则

奖励分稀有度：

```text
普通：白色
稀有：蓝色
史诗：紫色
传说：金色
```

### 1. 普通奖励

普通奖励只能是单个字：

```text
获得字：万
获得字：箭
获得字：固
获得字：若
获得字：画
```

如果该字已经拥有，则该字副本 +1。

普通奖励不直接给完整成语。

### 2. 稀有奖励

稀有奖励可以是小构筑强化：

```text
战斗袋容量 +1
成语字池容量 +2
补字速度 +10%
预览字数量 +1
城门恢复 2 点血
百步穿杨伤害 +20%
下一批字中优先补缺字
```

### 3. 史诗奖励

史诗奖励影响构筑质量：

```text
复制一个已拥有字
移除一个字副本
战斗袋容量 +2
成语字池容量 +4
某个已解锁成语效果 +30%
补字速度 +20%
```

### 4. 金色奖励

只有金色奖励才可以直接给完整成语：

```text
获得成语：万箭齐发
立即加入：万、箭、齐、发
立即解锁：万箭齐发
```

金色成语奖励相当于：

```text
完整四字包 + 成语技能解锁
```

如果其中某些字已经拥有，则对应字副本继续 +1，不浪费。

## 十、成语解锁规则

建议采用：

```text
集齐某个成语的 4 个字后，自动解锁该成语
```

例如玩家通过普通奖励陆续获得：

```text
万、箭、齐、发
```

则自动提示：

```text
已集齐：万箭齐发
解锁成语：万箭齐发
```

金色奖励的价值在于：

```text
不需要慢慢集齐，直接四字加入并解锁
```

## 十一、字池容量和战斗袋容量限制

### 1. 必须限制成语字池

如果无限加字，后期会变成：

```text
百、步、穿、杨、万、箭、齐、发、固、若、金、汤、画、地、为、牢、火、上、浇、油……
```

结果是玩家想凑什么都很难。

建议第一版：

```text
成语字池容量：12
```

### 2. 必须限制战斗袋

战斗袋过大，玩家容易一次性凑太多技能；战斗袋过小，玩家容易卡死。

建议第一版：

```text
战斗袋容量：6
```

### 3. 容量可以作为奖励成长

后续奖励可以加入：

```text
成语字池容量 +2
战斗袋容量 +1
战斗袋容量 +2
预览字数量 +1
补字速度 +10%
```

这样“容量成长”会成为肉鸽构筑的重要一环。

## 十五、当前结论

当前项目不再继续旧版“多短关卡”方向。

新的核心方向是：

```text
强关卡
少关卡
单局 5~10 分钟
击杀升级
暂停三选一奖励
字池 / 战斗袋 / 成语槽三层流转
普通奖励给字
金色奖励给完整成语
成语无冷却
用字流转速度控制释放频率
```

这套方向比旧版更有差异化，也更能体现“成语塔防”的核心卖点。
