# v0.6 完整工程系统（上线级架构设计）

> 目标：将 v0.5.2 的可运行架构升级为「可上线、可扩展、可编辑内容驱动系统」

---

# 一、v0.6 核心目标

v0.6 的本质不是“加功能”，而是三件事：

## 🎯 1. 系统规则化（Rule-driven）
- Boss 不再是单位，而是“规则修改器”
- 敌人不再是数值，而是“行为节点”

## 🎯 2. 内容数据化（Data-driven）
- 成语
- 字池
- 技能
- Boss
全部可配置

## 🎯 3. 编辑器化（Tool-driven）
- 可视化编辑关卡
- 可配置敌人行为
- 可调节成语效果

---

# 二、AI敌人系统（Behavior System 2.0）

## 2.1 敌人从“类型”升级为“行为组件”

### ❌ v0.5
- 盾兵
- 弓兵
- 骑兵

### ✅ v0.6
- MoveComponent
- AttackComponent
- SupportComponent
- DisruptLetterComponent

---

## 2.2 行为组件结构

```csharp
public interface IEnemyBehavior
{
    void Tick(EnemyContext ctx);
}
```

---

## 2.3 示例：字池干扰AI

```csharp
public class DisruptLetterBehavior : IEnemyBehavior
{
    public void Tick(EnemyContext ctx)
    {
        if (ctx.playerNear)
        {
            ctx.letterSystem.InjectNoise();
        }
    }
}
```

---

## 2.4 敌人 = 组件组合

```csharp
Enemy = [Move + Attack + Disrupt]
Enemy = [Move + Tank]
Enemy = [Move + Support + Heal]
```

# 三、Boss规则引擎（核心升级点）

## 3.1 Boss不再是单位，而是“规则插件”

---

## 3.2 Boss接口

```csharp
public interface IBossRule
{
    void OnWaveStart(GameContext ctx);
    void OnLetterSpawn(GameContext ctx);
    void OnIdiomCast(GameContext ctx);
}
```

---

## 3.3 示例Boss：乱字妖师

```csharp
public class ChaosBossRule : IBossRule
{
    public void OnLetterSpawn(GameContext ctx)
    {
        ctx.letterSystem.ShuffleQueue();
    }
}
```

---

## 3.4 示例Boss：火城督军

```csharp
public class FireBossRule : IBossRule
{
    public void OnWaveStart(GameContext ctx)
    {
        ctx.map.EnableFireZones();
    }
}
```

---

## 3.5 Boss系统本质

> Boss = 运行时规则注入器（Runtime Modifier）
