# ADR-0001：保留横屏战场并采用短房间状态机

## 状态

已接受

## 背景

当前 Demo 使用 1280×720 横屏，敌人从左侧推进到右侧城门。横向距离承载了普通兵、盾兵、骑兵、弓兵的站位差异，也是城门压迫感和范围技能表现的基础。

现有单局由一个包含 180 个敌人的连续长波次组成，缺少清晰的小胜利、房间奖励和正式胜利结算，不符合《成语塔防》爆款化方案要求的 4～6 分钟短局与高频成长。

## 决策

保留 1280×720 横屏基准和“左侧刷怪、中央交战、右侧城门”的战场关系。

将单一长波次替换为房间状态机：

```text
PREPARE → COMBAT → CLEARING → REWARD → NEXT_ROOM
                                 ↓
                              VICTORY
```

P0 使用 3 个普通房间和 1 个 BOSS 房验证闭环；验证通过后再扩展为 5 个普通房间和 1 个 BOSS 房。

## 结果

### 正面

- 保留现有战场资产、敌人移动和技能表现。
- 每 35～75 秒产生一次房间完成和成长反馈。
- 房间配置可数据化，便于调整节奏。
- 胜利、失败、奖励暂停的状态边界更明确。

### 负面

- `WaveSystem` 需要从单波计时器升级为房间推进系统。
- `SystemManager` 需要区分刷怪完成与房间真正清空。
- 必须补充胜利结算和房间切换验证。

### 中性

- “房间”是节奏单位，不代表切换地图；首版仍可复用同一守城场景。

## 备选方案

### 保持 180 敌人连续刷怪

未采用。改造成本低，但不能形成短房间奖励和清晰阶段目标。

### 改为竖屏

未采用。会压缩敌人行军距离、弓兵站位和城门空间，削弱塔防压迫感。
