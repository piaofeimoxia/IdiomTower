# 《成语塔防》爆款化产品改造方案

> 文档版本：v1.0  
> 日期：2026-06-25  
> 目标平台：抖音小游戏  
> 核心方向：低理解成本、高反馈密度、轻肉鸽成长、社交留存入口前置

---

## 一、改造结论

《成语塔防》不需要推翻“拼成语释放技能”的核心创意，需要改变的是承载这个创意的产品结构。

当前版本更像一个用于验证战斗机制的横屏 Demo：

- 玩家需要拖动字块拼成语。
- 单局采用长时间连续刷怪。
- 肉鸽奖励已经发展到“五选二”。
- 信息主要通过文字 HUD 和临时提示呈现。
- 游戏开始后直接进入战斗，局外成长、社交回流和商业化节点较弱。

目标版本应改造成一款横屏、低操作负担、短房间推进的抖音小游戏：

> 玩家只需要点字组成成语，几秒释放一次技能；敌军在横向战场持续压向城门，每完成一个短房间就获得一次明显成长，几分钟完成一轮守城，并在变强、失败和领取奖励时自然接触视频分享、激励视频与回流入口。

推荐采用“保留局内核心，重做内外循环”的中度改造路线。

不建议只在现有 Demo 上增加广告、分享和签到入口。现有局内节奏还没有形成足够强的短反馈，直接叠加运营入口会让产品显得拥挤。也不建议重做成纯答题或纯合成游戏，这会丢失“成语是技能”的差异化卖点。

---

## 二、目标用户体验

玩家第一次进入游戏时，应在 10 秒内完成以下认知：

1. 敌人正在攻城。
2. 点击四个字可以组成成语。
3. 成语会自动变成技能攻击敌人。
4. 打完当前房间就能变强。

第一局前 60 秒内必须出现：

- 至少 3 次成语释放。
- 至少 1 次成片击杀或强反馈。
- 至少 1 次升级选择。
- 至少 1 次“差一点失败但成功解决”的压力变化。

玩家完成首局后应形成三个记忆点：

- “我会玩”：操作只有点字、撤回和选奖励。
- “很爽”：成语真的会变成箭雨、护盾、火焰或控制。
- “还能更强”：下一局能带着新成语、局外升级或好友助力继续挑战。

## 三、产品结构调整

### 3.1 保留横屏战场，从连续长波次改为短房间

保留当前 1280×720 横屏比例及“敌人从左侧进攻、城门位于右侧”的主要空间关系。

横屏是本项目玩法表达的一部分，不应为了套用常见小游戏模板而改成竖屏。敌人的行军距离、不同兵种的前后关系、弓兵远程站位、成群敌人与城门危机都需要足够的横向空间。竖屏会压缩战场宽度，让敌人、技能和城门挤在一起，削弱塔防压迫感。

横屏需要改变的不是方向，而是节奏：把目前 180 个敌人连续生成的长波次拆成多个短房间。

一局由 6 个短房间组成：

| 阶段 | 时长 | 主要作用 |
|---|---:|---|
| 房间 1 | 35~45 秒 | 教玩家点字施法，保证第一次成功 |
| 房间 2 | 40~50 秒 | 加入盾兵，第一次奖励选择 |
| 房间 3 | 45~55 秒 | 加入骑兵，制造速度压力 |
| 房间 4 | 45~55 秒 | 加入弓兵或精英，检验防御能力 |
| 房间 5 | 50~60 秒 | 高密度敌群，检验范围技能 |
| BOSS 房 | 60~75 秒 | 制造翻盘和传播素材 |

目标单局时长控制在 4~6 分钟。房间结束时立即暂停，发放奖励并进入下一房间。玩家即使中途退出，也能明确知道自己刚刚完成了什么。

房间制的价值不只是缩短时长。它把一局拆成多个可感知的小胜利，使“战斗—奖励—变强—再战斗”的循环频率显著提高。

### 3.2 从拖拽拼字改为低负担点字

主操作改为：

- 点击战斗袋中的字，自动进入第一个空槽。
- 点击成语槽中的字，撤回战斗袋。
- 四个槽填满后自动判断。
- 拼对立即释放技能。
- 拼错时不惩罚生命，只轻微震动并提示可撤回。

拖拽可以作为辅助操作保留，但不能再作为唯一操作。横屏模式下玩家可能双手持机，也可能把手机横放，核心要求不是强行单手，而是减少手指移动距离，让玩家点击四次即可完成一次施法。

前两局增加弱提示：

- 可组成的字产生轻微呼吸高亮。
- 玩家停顿 3 秒后，高亮第一个正确字。
- 连续拼错两次，直接显示一个成语轮廓。
- 开局第一组成语固定保证可拼。

目标不是考倒玩家，而是让玩家快速完成“看懂—点击—施法—爽”的闭环。

## 四、新核心循环

### 4.1 单个房间循环

```text
进入房间
→ 敌人快速出现
→ 玩家点字组成成语
→ 成语自动释放技能
→ 获得击杀、经验和掉落反馈
→ 30~60 秒结束房间
→ 领取奖励或三选一强化
→ 进入下一房间
```

### 4.2 单局循环

```text
选择一个初始成语或好友助力
→ 连续完成 5 个普通房间
→ 每 1~2 个房间获得一次构筑选择
→ 挑战 BOSS
→ 结算铜钱、残卷和宝箱
→ 解锁或升级局外能力
→ 分享战绩、领取翻倍、开始下一局
```

### 4.3 长线循环

```text
每日登录领取资源
→ 完成一局获得残卷
→ 解锁新成语或提升成语等级
→ 完成每日任务和好友协作
→ 挑战更高章节或更高难度
→ 回到主页领取离线资源、信件和活动奖励
```

## 五、降低理解成本

### 5.1 首页只保留一个主目标

首页中央只放一个高优先级按钮：

> 开始守城

按钮旁显示本次可获得的主要奖励，例如“再过 2 关解锁万箭齐发”。其他入口围绕主按钮布置，但不与主按钮竞争：

- 左上：头像、等级、战力。
- 右上：设置、邮件。
- 左侧：每日任务、好友助力。
- 右侧：福利、活动、快捷回流入口。
- 底部：成语、城门、排行三个长期入口。

抖音小游戏的首页还应承担“从内容进入游戏”的承接作用。玩家可能刚看完一条极限翻盘或全屏清怪的视频，因此首页不应先让玩家领取大量资源、阅读公告或配置复杂阵容，而应让玩家立刻复现视频里看到的玩法。

### 5.2 数值表达结果，不表达公式

奖励卡不展示复杂百分比叠乘和长描述，优先使用玩家能直接理解的结果：

- “箭雨多落一轮”
- “城门立即恢复 3 点血”
- “每次施法多获得 1 个字”
- “冻结时间变长”
- “百步穿杨可再穿透 1 人”

同一局最多出现三类核心资源：

- 城门生命。
- 战斗袋中的字。
- 当前经验进度。

其余信息折叠或在需要时出现。

### 5.3 控制初期成语数量

首日只重点出现 4 个高认知成语：

- 百步穿杨：单体穿透。
- 万箭齐发：全屏清怪。
- 固若金汤：城门护盾。
- 画地为牢：全场控制。

后续再逐步加入火系、召唤系和联动成语。第一局不让玩家同时理解多个流派，也不要求玩家阅读完整构筑说明。

## 六、提高反馈密度

反馈密度的目标不是让画面一直爆炸，而是让玩家每隔几秒都知道自己做对了什么。

### 6.1 反馈节奏标准

| 时间尺度 | 应出现的反馈 |
|---|---|
| 0.1 秒内 | 点字缩放、音效、轻震、槽位吸附 |
| 1 秒内 | 成语组成、大字横幅、技能起手 |
| 2 秒内 | 敌人受击、击杀数字、经验增长 |
| 3~5 秒 | 下一次可拼机会或明显战场变化 |
| 20~30 秒 | 小升级、宝箱、精英击杀 |
| 40~60 秒 | 房间完成、奖励选择、地图推进 |

### 6.2 成语释放的四层反馈

每次成功施法统一包含：

1. 成语四字在屏幕中央快速合并。
2. 技能名称出现 0.5 秒。
3. 战场技能动画与敌人受击同步发生。
4. 结果数字明确显示，例如“击破 8”“护盾 +5”“冻结 6 人”。

不同技能必须有不同的声音、颜色和画面运动方向。玩家不看文字也应能分辨箭雨、防御、火焰和控制。

### 6.3 制造可传播的翻盘瞬间

每局至少主动制造一次以下局面：

- 城门只剩少量生命时，战斗袋出现可拼的防御成语。
- BOSS 冲锋前，系统补入控制成语所需的最后一个字。
- 大量敌人聚集时，出现范围技能的完整组合。

这不是暗改胜负，而是用受控随机保证玩家能遇到“我靠一个成语救回来了”的传播时刻。

## 七、轻肉鸽成长改造

### 7.1 奖励从“五选二”改回“三选一”

现有五选二虽然选择更多，但增加了阅读、比较、反选和确认成本，不符合轻肉鸽定位。

建议标准奖励使用三选一，每张卡只有：

- 一个图标。
- 一个短标题。
- 一句结果描述。

选择后立即生效，不再增加确认步骤。

### 7.2 每局只形成一个明显流派

首版保留三个流派：

| 流派 | 核心体验 | 代表强化 |
|---|---|---|
| 箭术流 | 高频直接伤害 | 穿透增加、箭雨追加、暴击 |
| 守城流 | 护盾和恢复 | 护盾强化、低血减伤、修门 |
| 控制流 | 冻结和减速 | 冻结延长、控制增伤、群体减速 |

奖励生成时提高同流派连续出现的概率。玩家选过一次箭术强化后，后续更容易看到箭术相关选项，从而自然形成构筑，不要求玩家提前学习复杂套路。

### 7.3 强化分成三种

- 普通强化：立即提升一个明确效果。
- 成语进化：改变技能表现或机制。
- 奇遇奖励：通过宝箱、精英或激励视频获得额外选择。

成语进化应优先做“机制变化”，例如：

- 百步穿杨：穿透同排全部敌人。
- 万箭齐发：箭雨结束后补射最前方敌人。
- 固若金汤：护盾破裂时击退全体敌人。
- 画地为牢：冻结结束时造成一次伤害。

这种变化比单纯增加 10% 数值更容易被玩家感知。

## 八、成长与商业化结合

商业化只放在玩家明确产生欲望的节点，不使用无意义的战斗中断。

### 8.1 推荐激励视频节点

| 节点 | 玩家心理 | 激励内容 |
|---|---|---|
| 奖励选择后 | 想让构筑更完整 | 免费刷新一次三选一 |
| BOSS 前 | 担心当前强度不足 | 临时获得一个成语或强化 |
| 城门被破 | 不想放弃本局构筑 | 原地复活，恢复部分生命 |
| 通关结算 | 想放大奖励 | 铜钱或残卷翻倍 |
| 离线奖励 | 顺手领取 | 领取双倍离线收益 |
| 成语差一片解锁 | 解锁欲最强 | 获得一次额外残卷 |

每个节点只提供一个明确收益，不同时弹出多个广告入口。

### 8.2 资源设计

首版长期资源控制在三种：

- 铜钱：升级城门和基础能力。
- 成语残卷：解锁、升级成语。
- 体力或挑战次数：控制高价值关卡节奏，可在验证留存后再决定是否加入。

不要在首版同时引入军功、宝石、钥匙、装备材料等多套资源。

### 8.3 付费成长边界

商业化应加速成长、增加选择或挽救失败，但不能让玩家跳过“拼成语施法”这个核心过程。

不建议售卖自动战斗作为主要功能。可以提供弱辅助，例如：

- 首字提示。
- 开局多一个字。
- 每局一次自动补齐。

玩家仍然需要亲手完成成语，才能保留产品记忆点。

## 九、抖音传播与留存入口前置

### 9.1 首页制造“还有人在玩、还有内容可晒”的感觉

即使首版没有复杂多人玩法，也应提供轻量多人痕迹：

- 好友或同城玩家的最高房间记录。
- “有人刚刚用万箭齐发完成 1 血翻盘”的动态。
- 本周守城人数和今日挑战热度。
- 好友赠送的一次开局助力。
- 城门旁显示其他玩家留下的守城旗帜或弹幕式战绩。

这些信息的作用是降低单机孤独感，让玩家知道游戏存在持续回来的理由。

### 9.2 推荐传播与社交功能

第一阶段只做低成本、高感知功能：

1. 好友助力：每天可领取一次开局强化。
2. 战绩分享：生成“1 血翻盘”“本局最强成语”“一招击破多少敌人”等短视频或战绩卡素材。
3. 好友排行：比较最高房间、最高击杀或最快通关。
4. 求助残卷：差一片解锁成语时可向好友求助。
5. 收藏或快捷回流奖励：根据抖音小游戏实际可用能力配置明确且不过量的奖励。

传播内容应由游戏自动截取最强的 3~5 秒，而不是要求玩家自己录屏。优先记录：

- 城门低血量翻盘。
- 万箭齐发全屏清怪。
- BOSS 冲锋被画地为牢打断。
- 单次百步穿杨连续贯穿。
- 本局首次解锁新成语。

暂不做实时联机、公会战和复杂聊天系统。

### 9.3 回流入口

玩家每天回来时首页应有 2~3 个可立即领取的反馈：

- 离线守城收益。
- 好友赠礼或系统信件。
- 每日首次挑战奖励。

领取过程控制在 10 秒内，随后立刻引导进入一局，不让玩家在红点系统里停留太久。

## 十、首局流程

### 0~10 秒：不讲规则，直接行动

- 敌人出现并向城门移动。
- 战斗袋中固定出现“百、步、穿、杨”。
- 手指动画提示依次点击四个字。
- 百步穿杨自动释放并击杀敌人。

### 10~40 秒：重复一次并增加压力

- 玩家独立完成第二次施法。
- 敌人数量增加。
- 经验条快速增长。
- 房间结束，出现第一次三选一。

### 40~90 秒：展示差异化爽点

- 系统引导获得“万箭齐发”体验卡或临时技能。
- 敌人密集出现。
- 玩家完成万箭齐发，全屏清怪。

### 90~180 秒：制造第一次选择与悬念

- 玩家进入新房间。
- 出现盾兵或骑兵。
- 城门承受一次可见伤害。
- 玩家选择偏输出或偏防御的强化。

### 首局结束

无论胜负，都给出明确成长：

- 解锁一个成语残卷。
- 展示下一成语解锁进度。
- 提供结算翻倍或好友助力入口。
- 主按钮直接变成“再守一局”。

## 十一、横屏界面布局建议

保留当前横向战场，但重新分配 1280×720 的信息层级：

```text
顶部 12%：房间进度、经验、城门生命、暂停
中部 50%：左侧刷怪区 → 中央交战区 → 右侧城门
底部 38%：成语槽、战斗袋、弃字与提示
```

战场必须拥有最高视觉优先级。敌人从左向右推进的距离不能被首页按钮、常驻活动入口或大面积 HUD 挤占。战斗中的运营入口全部隐藏，仅保留暂停按钮。

底部操作区横向排列：

```text
左侧：当前可用字和补字预览
中间：四格成语槽
右侧：弃字、撤回和一次性提示
```

字块与成语槽之间的移动距离应尽量短。默认点击自动入槽，玩家不需要跨越整个屏幕拖动字块。技能释放时，成语文字可从底部槽位飞向中部战场，把操作区与战斗结果连接起来。

HUD 不再显示内部调试数据，例如刷新数量、字池总量、版本名和系统状态。开发信息统一放入调试面板。

## 十二、与当前代码的对应改造

### 12.1 可以保留的部分

- `GameBootstrap` 启动入口。
- `SystemManager` 的总体协调职责。
- `EnemySystem` 的敌人状态、伤害、冻结和弓兵机制。
- `WaveSystem` 的定时生成能力。
- `ViewSystem` 的贴图资源与敌人动画加载方式。
- 当前四个成语及其技能原型。
- 字池、战斗袋、弃字延迟回池的基础概念。

### 12.2 需要重做的部分

- `WaveSystem`：从单一 180 敌人长波次改为房间状态机。
- `SystemManager`：增加房间推进、胜利结算、局外奖励和受控随机。
- `ViewSystem`：保留横屏战场，重排 HUD 与底部操作区，并重做点字操作、奖励卡、首页和结算页。
- 奖励系统：从五选二改为三选一，并增加流派权重。
- 字块交互：默认点击入槽，拖拽降为辅助。
- 场景：保留 1280×720 横屏基准，增加不同横屏分辨率和安全区适配。

### 12.3 应逐步消除的技术风险

当前多个补丁文件通过修改原型覆盖同一方法，最终行为依赖导入顺序。继续增加玩法时，调试成本会迅速上升。

进入正式改造前，建议把已验证的补丁合并回正式类：

- 合并奖励选择逻辑。
- 合并百步穿杨穿透逻辑。
- 合并弃字回池逻辑。
- 合并弓兵表现与失败界面。

随后将核心配置拆成数据表，避免成语、敌人、奖励和关卡数值继续硬编码在 `SystemManager` 与 `ViewSystem` 中。

## 十四、核心验证指标

首轮测试不追求完整商业数据，优先验证玩法是否真正变轻、变爽。

### 新手理解

- 首次成功释放成语的中位时间低于 12 秒。
- 首局完成两次成语释放的玩家比例高于 85%。
- 首局因不会操作退出的比例低于 10%。

### 反馈与节奏

- 平均每 5 秒至少发生一次有效操作或强反馈。
- 玩家首局 60 秒内至少完成一次奖励选择。
- 首局平均施法次数不少于 8 次。

### 留存意愿

- 首局结束后点击“再来一局”的比例。
- 成语解锁进度页的查看率。
- 好友助力、视频分享、收藏和快捷回流入口点击率。

### 商业化

- 失败复活接受率。
- 奖励刷新接受率。
- 结算翻倍接受率。

如果“奖励刷新”明显高于“失败复活”，说明玩家更在意构筑；如果“失败复活”更高，说明局内投入感更强。两者都能用于反推后续内容重点。

---

## 十五、明确不做的内容

首个爆款验证版本暂不做：

- 实时多人协作。
- 公会和复杂社交关系链。
- 装备、角色、宠物等多套养成。
- 十种以上流派。
- 过多资源和付费货币。
- 长剧情和大量对话。
- 需要背诵生僻成语的高难答题。
- 战斗中强制插屏广告。

产品必须先证明一件事：

> 玩家是否愿意因为“点四个字就能释放一个夸张技能”而连续玩第二局。

## 十六、最终产品表达

改造后的《成语塔防》不再被描述为“成语答题加塔防”，而应统一表达为：

> 点字放大招，三分钟守住城。

核心卖点文案：

- 别人靠兵守城，我靠成语施法。
- 城门只剩 1 点血，一个成语还能救回来。
- 点四个字，全屏敌人一起消失。
- 每过一个房间，都能让成语再进化一次。

最终体验结构应稳定为：

```text
低门槛点字
→ 高频技能反馈
→ 短房间胜利
→ 轻量随机成长
→ 失败挽救或奖励翻倍
→ 好友痕迹与回流领取
→ 再来一局
```

这套改造保留了《成语塔防》最有辨识度的横向守城战场，同时把它装进更适合抖音小游戏短视频传播、短局留存和商业化的产品框架中。
